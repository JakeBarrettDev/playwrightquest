## Mental model before touching any code

The test container needs to reach the Next.js server to access the fictitious site (e.g. `http://localhost:3000/sites/bramble-co/`). On local development, the container can reach the host via `host.docker.internal`. In production, it's the deployed URL. This is controlled by the `PQ_BASE_URL` env var that already exists in `playwright.config.ts`. **The DockerRunner must pass `PQ_BASE_URL` through to the container.**

The official Playwright Docker image (`mcr.microsoft.com/playwright:v1.59.1-noble`) ships with Node.js and all browser binaries pre-installed. We do not need a custom Dockerfile. We run `docker run` with the image, mount a temp directory containing the test file and config, and stream stdout/stderr back to the Next.js process.

## Part 3 ‚Äî Implement DockerRunner

**File: `lib/execution/dockerRunner.ts`**

Replace the stub with a full implementation. Here is the complete algorithm:

### 3a. Constructor
```typescript
constructor(private projectRoot: string) {}
```
Accept `projectRoot` (same as LocalRunner) so `index.ts` can pass it consistently.

### 3b. `run(request)` method

**Step 1 ‚Äî Create a temp directory**
Use `fs.mkdtemp` under `os.tmpdir()` with prefix `pq-docker-`. Call it `runDir`.

**Step 2 ‚Äî Write test file**
Write `request.code` to `${runDir}/player.spec.ts`.

**Step 3 ‚Äî Write a JavaScript playwright config** (not TypeScript ‚Äî no transpilation needed inside the container)

Write `${runDir}/playwright.config.js` with this content, substituting `PQ_BASE_URL` from `process.env.PQ_BASE_URL ?? 'http://host.docker.internal:3000'`:

```javascript
// Auto-generated by DockerRunner ‚Äî do not edit
module.exports = {
  testDir: '/work',
  testMatch: ['player.spec.ts'],
  timeout: 20000,
  expect: { timeout: 5000 },
  workers: 1,
  retries: 0,
  reporter: [['line']],
  use: {
    baseURL: '${PQ_BASE_URL}',
    headless: true,
    screenshot: 'only-on-failure',
  },
  projects: [{ name: 'chromium', use: { browserName: 'chromium' } }],
  outputDir: '/work/test-results',
};
```

If `request.browser` is `firefox` or `webkit`, adjust `browserName` accordingly.

**Step 4 ‚Äî Spawn docker run**

```
docker run --rm \
  --network host \                          # Mac/Linux: lets container reach host.docker.internal
  -v ${runDir}:/work \
  -e PQ_SITE=${request.site} \
  -e PQ_BASE_URL=${baseUrl} \
  -e CI=1 \
  mcr.microsoft.com/playwright:v1.59.1-noble \
  npx playwright test /work/player.spec.ts --config /work/playwright.config.js
```

Use `child_process.spawn('docker', [...args])` ‚Äî do NOT use shell: true.

**Important network note:** On Linux, `--network host` lets the container reach `localhost` directly. On macOS with Docker Desktop, `host.docker.internal` resolves automatically without `--network host`. Detect the platform: if `process.platform === 'linux'`, add `--network=host` and use `http://localhost:3000` as baseURL; otherwise use `host.docker.internal:3000`. But always allow override via `process.env.PQ_BASE_URL` ‚Äî that takes highest priority.

**Step 5 ‚Äî Stream output**

Pipe `proc.stdout` and `proc.stderr` both to a single output handler. Split on newlines and call `request.onOutput?.(line)` for each non-empty line. Also accumulate into `stdoutBuf` and `stderrBuf` strings (same as LocalRunner).

**Step 6 ‚Äî Timeout handling**

Same pattern as LocalRunner: `setTimeout` at `(request.timeoutMs ?? 30_000) + 5_000`. On timeout, call `proc.kill()` and resolve with a timed-out `ExecutionResult`.

**Step 7 ‚Äî On process close**

After process exits:
- Read screenshots: glob `${runDir}/test-results/**/*.png`, take the first one, read to base64
- Parse error message from stdout/stderr (extract lines starting with `Error:` or containing `TimeoutError`, same logic as LocalRunner)
- Clean up: `rm(runDir, { recursive: true, force: true })` in a `finally` block
- Resolve with `ExecutionResult`

### 3c. Error if Docker is not available

Before spawning, check that `docker` is on PATH by running `docker --version` synchronously with `spawnSync`. If it fails, reject with a clear error: `"Docker is not available. Install Docker Desktop or set EXECUTION_RUNNER=local."`

## Part 5 ‚Äî Streaming API route

**File: `app/api/execute/route.ts`**

Change the response from `Response.json(result)` to a streaming `text/event-stream` response.

### Event format

Each event is a newline-delimited JSON object followed by `\n\n`:

```
data: {"type":"output","line":"Running 1 test using 1 worker"}\n\n
data: {"type":"output","line":"  ‚úì  1 player.spec.ts:5:3 ‚Ä∫ checkout happy path (3.2s)"}\n\n
data: {"type":"result","result":{...ExecutionResult...}}\n\n
```

Two event types:
- `{ type: "output", line: string }` ‚Äî emitted as output arrives
- `{ type: "result", result: ExecutionResult }` ‚Äî emitted once at the very end

### Implementation

```typescript
export async function POST(request: Request) {
  const body = await request.json();
  // ... validate body (same as before) ...

  const encoder = new TextEncoder();
  const stream = new ReadableStream({
    async start(controller) {
      const enqueue = (obj: object) =>
        controller.enqueue(encoder.encode(`data: ${JSON.stringify(obj)}\n\n`));

      const runner = createRunner(process.cwd());
      const result = await runner.run({
        code: body.code,
        site: body.site,
        challengeId: body.challengeId,
        timeoutMs: body.timeoutMs,
        browser: body.browser,
        onOutput: (line) => enqueue({ type: 'output', line }),
      });

      enqueue({ type: 'result', result });
      controller.close();
    },
  });

  return new Response(stream, {
    headers: {
      'Content-Type': 'text/event-stream',
      'Cache-Control': 'no-cache',
      'Connection': 'keep-alive',
    },
  });
}
```

Handle errors inside the stream `start` by enqueuing `{ type: 'error', message: string }` and closing.


Reset `outputLines` to `[]` at the start of `handleRun`.

### 6b. Replace the fetch block

```typescript
const res = await fetch('/api/execute', {
  method: 'POST',
  headers: { 'content-type': 'application/json' },
  body: JSON.stringify({ code, site: challenge.site, challengeId: challenge.id }),
});

if (!res.ok || !res.body) {
  setExecError(`HTTP ${res.status}`);
  setRunning(false);
  return;
}

const reader = res.body.getReader();
const decoder = new TextDecoder();
let buffer = '';
let execution: ExecutionResult | null = null;

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  buffer += decoder.decode(value, { stream: true });
  const events = buffer.split('\n\n');
  buffer = events.pop() ?? '';
  for (const event of events) {
    const line = event.replace(/^data: /, '').trim();
    if (!line) continue;
    try {
      const msg = JSON.parse(line);
      if (msg.type === 'output') {
        setOutputLines(prev => [...prev, msg.line]);
      } else if (msg.type === 'result') {
        execution = msg.result;
        setExecResult(msg.result);
      } else if (msg.type === 'error') {
        setExecError(msg.message);
        setRunning(false);
        return;
      }
    } catch { /* malformed chunk, skip */ }
  }
}

if (!execution) {
  setExecError('No result received from runner');
  setRunning(false);
  return;
}
```

Continue into grading with `execution` exactly as before.
